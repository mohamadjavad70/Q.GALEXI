# گزارش بازآرایی معماری پروژه Q.GALEXI

## خلاصه اجرایی
این پروژه از حالت ریشه‌ی شلوغ و تخت به ساختار ماژولار مبتنی بر src منتقل شد. فایل‌های رابط کاربری، صفحات، هوک‌ها، کتابخانه‌ها، داده‌ها، اسکریپت‌ها و فایل‌های عمومی از هم تفکیک شدند.

## ساختار جدید
- src/pages: صفحات اصلی مسیرها
- src/components/ui: کامپوننت‌های پایه رابط کاربری
- src/components/galaxy: اجزای صحنه کهکشانی
- src/components/solarsystem: اجزای ناوبری و HUD منظومه
- src/components/stars: ابزارهای ستاره‌ها
- src/hooks: هوک‌های سفارشی
- src/lib: منطق کمکی و موتورهای داخلی
- src/data: داده‌ها و رجیستری‌ها
- src/test: تست‌ها
- public: دارایی‌های استاتیک
- docs: مستندات و گزارش‌ها
- scripts: اسکریپت‌های کمکی
- archive/legacy: فایل‌های تکراری یا بکاپ

## تغییرات کلیدی اعمال‌شده
1. انتقال کامل سورس‌ها به درخت src
2. جداسازی صحیح page و component
3. انتقال فایل‌های استاتیک به public
4. آرشیو کردن فایل‌های تکراری و بکاپ
5. افزودن دستورالعمل workspace برای چارچوب شورای نور
6. اصلاح خطاهای اصلی lint و ناسازگاری importها

## راستی‌آزمایی انجام‌شده
- Build: موفق
- Test: موفق، 1 از 1 تست پاس شد
- Lint: بدون خطا، فقط warningهای غیرمسدودکننده باقی مانده‌اند
- IDE Errors: بدون خطا

## هشدارهای باقیمانده
warningهای باقی‌مانده عمدتاً مربوط به توصیه‌های React Fast Refresh و dependency array برخی hookها هستند و مانع build یا اجرا نیستند.

## پیشنهاد گام بعدی
- شکستن bundle اصلی برای کاهش حجم فایل نهایی
- انتقال helperهای exported از برخی فایل‌های UI به ماژول‌های جدا
- بهبود warningهای hook برای سکوت کامل lint
